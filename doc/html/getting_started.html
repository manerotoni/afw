Getting started





Main window overview:


The graphical user interface contains a menu bar, toolbars for
sorting/ranking, display options and loading data. There is a separate
toolbar at the bottom for navigation within a cellh5 file.
 There are dockwidgets (sidebars)  for Annotation, Sorting and
Contrast enhancement which supply most of the functionallity. All
toolbars docks can be toogled using the View menu. The central
widget is the gallery/ thumbnail viewer in the middle oft main window.

A progress bar
 in the status bar of the main window will appear
 beside an abort button.  Loading of data causen
heavy IO of the hard disk and may block the GUI unit loading is
finished.  


CellAnnotators own hdf data model is optimized for fast loading. Size
of the thumbnails is predefined and all thumbnails in the files will be
loaded. In contrast, Cellh5 does not support thumbnails, they are
generated during the loading process, which is time consuming. Size of
the gallery images is alterable and the number auf items that are
loaded can be changed (default is 500), since Cellh5 is a storage
formate for HTS-data. 

















The gallery- or thumbnail viewer is the central part of CellAnnotator.
Its purpose is to display all forground  or cell objects to
the user . The gallery viewer is not displaying a stiched image, it is
displaying a real thumbnail gallery where each image is a clickable and
selectable item.  After loading the item are arranged
arbitraily in the order the appear in the data file.

Quick Sort Items:


Sorting occurs linearly from the upper left to the bottom right corner.
It is necessary to define at least one reference example for sorting.
After
sorting the reference example will be positioned always in the upper
left
corner of the view. CellAnnotator uses by default the cosine
similarity a measure. If multiple items are
selected,  the arithmetic
mean is used  as reference example.
